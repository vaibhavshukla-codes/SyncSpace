<-------------------------WebRTC Architectures------------------->
1: P2P (Peer To Peer)
2: SFU (Selective Forwarding Unit)
3: MCU (Multi Point Control Unit)

1: P2P:(1:1 calls)
->In P2P Architecture each computer will be connected to other computers of P2P connection.
->We use P2P Architecture when there are less number of computers are connected each other using P2P connection.
->In P2P Architecture a single node(computer) can get multiple request so at that time computer might not be sufficient
  to handle it therefore we use other Architectures.
-> n(nodes) uploads and n(nodes) downloads.

2: SFU:(Group calls)
->In SFU each node(computer) will send it's data to the central server and that server will send the data of other nodes to each node.
->Server holds upload load and client holds download load.
->1 upload and n(nodes) download.

Inshort: Single upload and multiple downloads.(for each node)

3: MCU:(Large conferences)
->Each node(computer) sends data to the server and server collects each node data and combine it then returns combined single stream   (data) to each node.
-> Server holds maximum load.
-> 1 upload and 1 download.

Inshort: Single upload and single download.(for each node)
